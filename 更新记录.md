# 更新记录

这个文件放在 `README.md` 同级目录，用来记录站点每次修改后的内容，方便回看“改了什么、改了哪些文件、有没有验证通过”。

## 使用规则

- 每次改动后，按日期新增一条记录，不覆盖旧内容。
- 每条记录尽量写清楚 4 件事：修改主题、涉及文件、具体变化、验证结果。
- 如果只是新增文章或笔记，也建议记录来源目录、导入文件名和是否公开。
- 如果改动还没有提交到 GitHub，请明确写成“当前工作区，未提交”。

## 2026-03-17（当前工作区，未提交）

### 主题：移动端稳定性与静态部署兼容性整理

### 涉及文件

- `app/layout.tsx`
- `next.config.js`
- `app/search/page.tsx`
- `components/SearchPageClient.tsx`
- `components/Footer.tsx`
- `data/security/vulnhub-notes.mdx`
- `data/siteMetadata.js`
- `package.json`
- `yarn.lock`

### 修改内容

- 移除了首页运行时注入的 `Vercel Analytics` 组件，减少页面额外外链请求，降低移动端网络抖动时的影响。
- 收紧了 `CSP` 白名单，去掉当前未使用的外部脚本和 iframe 域名，降低页面对第三方域名的依赖。
- 把 `/search` 重构成“服务端预生成搜索索引 + 客户端读取 `q` 参数筛选”的形式，并补上 `Suspense`，让搜索页可以兼容静态导出。
- 新增 `components/SearchPageClient.tsx`，单独承接搜索结果页的客户端筛选逻辑。
- 重写了页脚社交图标的渲染方式，修复 `SocialIcon` 的类型问题，并整理页脚文案。
- 移除了 `data/security/vulnhub-notes.mdx` 中一张外部图床图片，避免移动端打开文章时卡在第三方图片加载。
- 清理了 `data/siteMetadata.js` 的站点元数据文案，保留当前线上地址配置。
- 从依赖里移除了 `@vercel/analytics`。

### 额外视觉调整

- `components/HomeProfileHero.tsx`
- `components/HomeSiteStats.tsx`

这两个文件继续做了首页 Hero 仪表盘增强：

- 首页左侧头像主视觉更大，阴影和卡片层次更明显。
- 右侧 `Blog Stats` 从简单数字卡扩展成“数字卡 + 内容分布 + 站点状态”组合。
- GitHub 按钮仍保留在头像下方居中。

### 验证结果

- `yarn build`：通过
- `EXPORT=1 yarn build`：通过

### 说明

- 这一轮修改已经把项目整理成“更适合纯静态托管”的状态。
- 但如果公开域名继续使用 `vercel.app`，中国大陆手机网络访问仍可能不稳定；这属于部署链路问题，不是页面代码本身完全能解决的。
