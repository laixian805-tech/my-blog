# UI 文案规则

## 核心约束
- 页面中不要出现给开发者看的说明性小字。
- 页面中不要出现“先看什么”“这里单独放”“首页更像入口地图”“内容待补充”“持续整理长期沉淀”这类解释型、引导型、自述型文案。
- 页面只展示面向访客的必要信息、标题、导航和动作。

## 说明迁移规则
- 开发说明、设计意图、维护备注统一写在本地文档中，不直接出现在页面。
- 配置文件中的字段如果只是为了说明实现意图，不应默认渲染到 UI。

## 列表页规则
- 文章类列表默认只展示标题和日期。
- 摘要、标签、次级动作属于 hover 或次级信息层。
- 不为归档页额外增加解释型页头大卡片。

## 首页规则
- 首页保持单屏主版面。
- `课程学习` 仅作为首页左下角入口存在，不再扩展为独立说明版块。
